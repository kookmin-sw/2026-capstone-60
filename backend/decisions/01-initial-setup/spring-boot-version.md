# 기술 결정: Spring Boot 버전

관련 이슈: #3 Spring Boot 프로젝트 초기 환경 세팅

## 결정: Spring Boot 4.0.x

## 선택지 (2026.03 기준, Spring Initializr 제공 안정 버전)
- Spring Boot 4.0.5 (OSS 지원: 2026.12까지)
- Spring Boot 3.5.13 (OSS 지원: 2026.06까지)
- 3.4 이하는 이미 지원 종료, Spring Initializr에서도 제외됨

## 비교

| 기준 | 3.5.x | 4.0.x |
|------|-------|-------|
| OSS 지원 종료 | 2026.06 | 2026.12 |
| 코드 작성 방식 (Controller, Service, JPA 등) | 동일 | 동일 |
| 내부 엔진 | Jackson 2, Hibernate 6 | Jackson 3, Hibernate 7 |
| Spring Initializr 기본 제공 | O | O (기본 선택) |

## 핵심 근거

1. 지원 기간: 3.5는 2026.06 종료로 프로젝트 일정(6월 중순)과 겹치며, 이후 확장 가능성을 고려하면 2026.12까지 지원되는 4.0이 안전하다.

2. 레퍼런스 차이 없음: 3.x → 4.x에서 바뀐 건 내부 엔진(Jackson, Hibernate 버전)이며, 개발자가 직접 작성하는 코드(어노테이션, API 구조, JPA 사용법 등)는 동일하다. 3.x 기반 자료를 4.0에서 그대로 적용할 수 있다.

3. 새 프로젝트: 기존 코드가 없어 마이그레이션 비용이 0이다. 시작 시점의 최신 안정 버전을 선택하지 않을 이유가 없다.
